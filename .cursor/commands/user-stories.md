# /user-stories

## Description
使用者故事 - 將需求拆解為個別小單元的實作任務

## Usage
將需求拆解為個別小單元的實作任務，每個任務都有明確的輸入輸出、驗收條件和依賴關係

## Prompt Template
```
請你幫我根據以下脈絡，整理為個別小單元的實作任務：

- 需求： @{需求文件}
- 模組拆解：@{模組拆解文件}
- 解決方案： @{解決方案文件}

任務請依照以下原則來生成，每一項任務不要太大：

- 每個任務都要能夠獨立完成，需要有明確的：
  - **輸入格式**：需要什麼樣的輸入資料？
  - **輸出格式**：會產生什麼樣的輸出結果？
  - **驗收條件**：什麼情況下才算完成？
  - **依賴關係**：需要先完成哪些任務？
- 任務之間的依賴關係要明確，可以參考模組拆解的依賴關係。

每個任務請幫我存成個別獨立的文件，文件名稱為「{任務名稱}.md」，方便後續追蹤和管理。
```

## User Story Components
- **標準格式**：「作為...我想要...以便...」
- **驗收條件**（Checklist）
- **優先級標記**
- **相關功能對應**

## Example
```
### US-002：查看「我們的日常」文章列表
**作為** 求職者
**我想要** 瀏覽「我們的日常」文章列表
**以便** 了解公司內部日常活動與文化

**驗收條件**：
- [ ] 區塊標題顯示「我們的日常」與「Career News」
- [ ] 每篇文章顯示標題、發布日期、標籤徽章
- [ ] 每篇文章前有藍色箭頭圖示，表示可點擊
- [ ] 文章列表垂直排列，每篇文章間有適當間距
- [ ] 預設每頁顯示 3 篇文章
- [ ] 標題可點擊，連結至文章詳情頁
- [ ] 日期格式為 YYYY-MM-DD
- [ ] 響應式設計在各種裝置上正常運作

**優先級**：P0
**相關功能**：文章列表區塊（2.1.2）
```
