# /test

## Description
測試驗證 - 確保功能實現都可以通過測試驗證

## Usage
為功能實作撰寫和執行測試案例，確保程式碼品質和功能正確性

## Two Testing Strategies

### Strategy 1: 單一任務測試驗證 (Single Task Testing)
專注於特定任務的測試驗證

#### Prompt Template
```
直接根據 @{任務} 撰寫測試案例，確保功能的實現都可以通過測試驗證。
```

### Strategy 2: 總體驗證 (Comprehensive Testing)
涵蓋多個面向的完整測試驗證

#### Prompt Template
```
請你為以下任務產生驗證案例，請參考以下規格：

- 需求文件：{需求文件路徑}
- 模組拆解：{模組拆解文件路徑}
- 解決方案：{解決方案文件路徑}

請產生以下類型的驗證案例：
1. 正常流程驗證：驗證功能在正常情況下的運作
2. 邊界案例驗證：驗證極端值、空值、無效輸入等情況
3. 錯誤處理驗證：驗證錯誤情況下的處理機制
4. 整合驗證：驗證與其他模組的整合是否正常
5. 行為驗證：驗證功能是否符合使用者預期，是否符合需求使用者故事的描述。

最後，檢驗目前的所有測試檔案，是否已經涵蓋這次生成的驗證案例。如果沒有，請補充相關的驗證案例。
```

## Test Case Types

### 1. 正常流程驗證 (Normal Flow Testing)
- 驗證功能在正常情況下的運作
- 確保主要使用場景都能正常執行
- 檢查預期輸出與實際輸出一致

### 2. 邊界案例驗證 (Edge Case Testing)
- 驗證極端值處理（最大值、最小值）
- 空值、無效輸入的處理
- 特殊字元或格式的處理

### 3. 錯誤處理驗證 (Error Handling Testing)
- 網路錯誤、API 失敗的處理
- 資料驗證錯誤的處理
- 異常狀況下的降級處理

### 4. 整合驗證 (Integration Testing)
- 與其他模組的整合是否正常
- 資料流程是否正確
- 介面相容性驗證

### 5. 行為驗證 (Behavioral Testing)
- 功能是否符合使用者預期
- 是否符合需求文件中的使用者故事描述
- UI/UX 體驗驗證
