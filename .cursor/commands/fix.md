# 程式碼錯誤修正指令 (Fix Code Issues)

## 編譯錯誤修正 (Compilation Errors)

### 1. 執行型別編譯檢查
```bash
npm run tsc -- --noEmit
```

### 2. 分析編譯錯誤
- 檢查錯誤訊息中的檔案路徑和行號
- 確認錯誤類型：型別錯誤、語法錯誤、模組匯入錯誤等

### 3. 修正編譯錯誤
- **型別錯誤**: 檢查變數型別宣告、函數參數型別、回傳型別
- **語法錯誤**: 檢查括號、逗號、分號等語法問題
- **模組錯誤**: 確認匯入路徑是否正確、檔案是否存在
- **介面錯誤**: 檢查介面定義和實作是否相符

## 測試錯誤修正 (Test Failures)

### 1. 執行測試並查看錯誤
```bash
npm test
```

### 2. 分析測試失敗
- **單元測試失敗**: 檢查測試邏輯、斷言條件、模擬物件設定
- **整合測試失敗**: 檢查組件間的互動、資料流、API 呼叫
- **快照測試失敗**: 確認是否為預期的 UI/資料變化

### 3. 修正測試錯誤
- 更新測試斷言以符合新的程式碼行為
- 修復被測試的程式碼邏輯錯誤
- 更新測試資料或模擬物件
- 處理非同步操作的測試時機問題

## 程式碼品質檢查 (Code Quality Checks)

### 1. 執行 Linting
```bash
npm run lint
```

### 2. 修正 Lint 錯誤
- 程式碼格式問題（縮排、空格、換行）
- 命名規範問題
- 未使用的變數/函數
- 程式碼複雜度問題

## 驗證修正 (Verification)

### 1. 重新執行所有檢查
```bash
npm run tsc -- --noEmit && npm test && npm run lint
```

### 2. 確認修正結果
- 編譯檢查通過 ✓
- 所有測試通過 ✓
- Linting 檢查通過 ✓
- 功能測試確認 ✓

## 特殊情況處理 (Special Cases)

### 依賴問題
- 檢查 `package.json` 依賴版本
- 重新安裝依賴：`npm install`
- 清除快取：`npm cache clean --force`

### 環境問題
- 檢查 Node.js 版本是否相符
- 確認環境變數設定正確
- 檢查資料庫連線或外部服務狀態

### 緊急修復
- 如果需要快速修復：直接編輯程式碼然後執行 `npm run fix`（如果有設定）
- 對於阻塞性錯誤：優先修復編譯錯誤，再處理測試錯誤
