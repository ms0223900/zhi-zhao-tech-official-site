# /requirement-planning

## Description
釐清歸納需求階段，依照設計圖或是需求文件擬出 PRD (產品需求文檔)

## Usage
將模糊需求轉換為具體明確的 PRD（產品需求文檔）

## Prompt Template
```
根據這個設計稿，幫我擬出需求 PRD 文檔
```

## PRD Document Structure
1. **需求概述**：背景、目標、目標用戶
2. **功能需求**：
   - 頁面、組件結構
   - 資料結構設計
   - 資料結構：TypeScript 介面定義
   - API 需求：API 端點與格式
3. **設計規範**：色彩、字體、間距、響應式
4. **技術實現**：組件結構、技術棧、資料獲取策略
5. **功能優先級**：P0（最優先）/P1/P2 分類
6. **驗收標準**：功能、效能、無障礙、瀏覽器相容性
7. **時程規劃**：預估開發時間、功能實作優先順序
8. **風險與考量**：技術、內容管理、安全性
