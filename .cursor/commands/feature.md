# /feature

## Description
實作功能 - 具體功能實作階段

## Usage
執行具體功能實作，包含前期需求分析與統整，以及功能實作的具體執行

## Related Activities
- 前期需求分析與統整：將比較模糊的需求變得更具體明確
- 功能實作執行：根據規劃進行具體的程式碼實作

## Analysis Prompt Template
```
請你幫我構想 {需求} 應該要可以實現什麼具體的功能？ 應該先設計哪些功能？
```

## Implementation Notes
- 參考既有需求文件和設計稿
- 確保功能實現符合使用者期望
- 遵循既定的技術架構和規範
- 保持程式碼品質和可維護性

## 透過以下步驟進行功能實作：
1. 最重要 - 分析需求
  - 確保需求文件和設計稿已經被更新，並且符合使用者故事的描述。
  - 先從測試案例、實作任務清單和實作程式碼，確保稍候要實作的功能「還沒實作」，再繼續往下執行。
2. 設計功能 
3. 生成測試案例（依照使用者故事的驗收條件，生成測試案例）
4. 測試功能
5. 實作功能
6. 驗收功能
7. 文件化功能
8. 更新需求文件
9. 更新使用者故事，確保使用者故事的驗收條件已經被滿足，並且符合使用者故事的描述。
