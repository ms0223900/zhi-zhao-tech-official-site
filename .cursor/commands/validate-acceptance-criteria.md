# /validate-acceptance-criteria

## Description
驗收條件檢查 - 系統性驗證使用者故事的驗收條件是否滿足

## Usage
執行使用者故事驗收條件的全面檢查，確保程式碼、測試和文件都符合需求規範

## Related Activities
- **程式碼檢查**：驗證功能實作是否正確且完整
- **測試檢查**：確保測試覆蓋率和測試品質
- **文件檢查**：確認相關文件是否更新且準確

## Implementation Notes
- **檢查順序**：建議按照「程式碼 → 測試 → 文件」的順序進行檢查
- **驗證標準**：每個驗收條件都應該有明確、可量化的驗證標準
- **測試優先**：優先確保測試通過，再檢查其他面向
- **文件同步**：功能實作完成後，立即更新相關文件
- **持續檢查**：驗收條件檢查應該在開發過程中持續進行，而非僅在最後階段

## Validation Commands
```bash
# 程式碼品質檢查
npm run lint          # ESLint 和 TypeScript 檢查
npm run build         # 建置驗證
npm run type-check    # 型別檢查（如果有獨立指令）

# 測試檢查
npm run test          # 執行所有測試
npm run test:coverage # 檢查測試覆蓋率

# 整合檢查
npm run ci           # 持續整合檢查（如果有設定）
```

## Best Practices
1. **明確定義**：每個驗收條件都應該具體、可驗證
2. **測試驅動**：先寫測試，再實作功能
3. **持續驗證**：在開發過程中頻繁檢查，而非等到最後
4. **團隊共識**：確保所有團隊成員都了解驗收標準
5. **自動化檢查**：盡可能將檢查流程自動化

## Common Issues & Solutions

### 問題：測試覆蓋率不足
**解決方案**：補充單元測試和整合測試，確保關鍵路徑都有測試

### 問題：文件過時
**解決方案**：將文件更新納入開發流程，功能完成後立即更新

### 問題：程式碼品質問題
**解決方案**：執行程式碼檢視，修復 Lint 錯誤，改善程式碼結構

### 問題：效能不符合要求
**解決方案**：進行效能測試，識別瓶頸，實施優化措施
