# User Stories - 人才專區「我們的日常」功能

## 文檔資訊

**專案名稱**：人才專區 - 我們的日常 (Career News)  
**文檔版本**：v1.0  
**建立日期**：2025-12-24  
**相關文檔**：[PRD-我們的日常-Career-News.md](./PRD-我們的日常-Career-News.md)

---

## 使用者角色定義

### 角色 1：求職者 (Job Seeker)
- **描述**：正在尋找工作機會的潛在求職者
- **目標**：了解公司文化、工作環境與員工生活
- **使用情境**：評估是否適合該公司文化

### 角色 2：現有員工 (Current Employee)
- **描述**：智兆科技的現職員工
- **目標**：查看公司活動、日常分享與最新消息
- **使用情境**：了解公司動態與參與活動

### 角色 3：訪客 (Visitor)
- **描述**：一般網站訪客
- **目標**：了解智兆科技企業文化與品牌形象
- **使用情境**：瀏覽公司資訊

---

## P0 優先級 - 核心功能（必須實現）

### US-002：查看「我們的日常」文章列表
**作為** 求職者  
**我想要** 瀏覽「我們的日常」文章列表  
**以便** 了解公司內部日常活動與文化

**驗收條件**：
- [x] 區塊標題顯示「我們的日常」與「Career News」
- [x] 每篇文章顯示標題、發布日期、標籤徽章
- [x] 每篇文章前有藍色箭頭圖示，表示可點擊
- [x] 文章列表垂直排列，每篇文章間有適當間距
- [x] 預設每頁顯示 3 篇文章
- [x] 標題可點擊，連結至文章詳情頁
- [x] 日期格式為 YYYY-MM-DD
- [x] 響應式設計在各種裝置上正常運作

**優先級**：P0  
**相關功能**：文章列表區塊（2.1.2）

---

### US-003：識別重要與最新文章
**作為** 求職者  
**我想要** 透過標籤徽章快速識別重要或最新文章  
**以便** 優先閱讀重要內容

**驗收條件**：
- [x] TOP 標籤顯示為橙色背景（#E57B42）
- [x] NEW 標籤顯示為白色背景，淺藍色框線，文字為淺藍色（#55BBF9）
- [x] 標籤為圓角按鈕樣式
- [x] 標籤位置在日期右側
- [x] 標籤文字清晰可讀

**優先級**：P0  
**相關功能**：標籤徽章（2.1.2.3）

---

### US-004：使用分頁功能瀏覽文章
**作為** 訪客  
**我想要** 使用分頁功能瀏覽多頁文章  
**以便** 有效率地查看所有文章

**驗收條件**：
- [x] 分頁功能正常運作，會顯示共 X 筆資料
- [x] 顯示當前頁碼（例如：第 1/2 頁）
- [x] 顯示頁碼按鈕（1、2、...、20）
- [ x] 當前頁碼高亮顯示（淺藍色背景）
- [x] 左右箭頭按鈕可切換頁面
- [x] 總頁數 ≤ 7 時顯示所有頁碼
- [x] 總頁數 > 7 時顯示首頁、末頁、當前頁及前後各 2 頁，中間用「...」省略
- [x] 顯示總筆數（「共 X 筆資料」）
- [x] 分頁切換流暢無卡頓

**優先級**：P0  
**相關功能**：分頁控制區塊（2.1.3）

---

### US-005：閱讀文章詳情
**作為** 求職者  
**我想要** 點擊文章標題進入詳情頁閱讀完整內容  
**以便** 深入了解公司活動與文化

**驗收條件**：
- [x] 點擊文章標題可進入詳情頁
- [x] 詳情頁顯示完整文章內容
- [x] 文章內容包含標題、副標題、發布日期、封面圖片、正文，可以直接沿用「複製」最新消息頁面的組件來使用
- [x] 有返回列表按鈕，可以返回上一頁（/careers）
- [x] 文章內容正確渲染（支援 Markdown 格式），文章內容 SCHEMA 同最新消息頁面
- [x] 響應式設計在各種裝置上正常顯示

**優先級**：P0  
**相關功能**：文章詳情頁（2.4.1）

---

## P1 優先級 - 重要功能

### US-006：調整每頁顯示數量
**作為** 現有員工  
**我想要** 調整每頁顯示的文章數量  
**以便** 根據個人偏好自訂瀏覽體驗

**驗收條件**：
- [x] 提供每頁顯示數量選擇器（下拉選單）
- [x] 預設值為 3 筆
- [x] 選項包含：3、6、9、12
- [x] 選擇後，按下「送出」按鈕，立即更新列表顯示
- [x] 分頁邏輯、總頁數和到第「X」頁會根據新設定重新計算
- [x] 當每頁顯示數量選擇器改變時，自動重置「到第 X 頁」輸入框的值

**優先級**：P1  
**相關功能**：每頁顯示數量選擇器（2.1.3.1）

---

### US-007：跳轉到指定頁碼
**作為** 訪客  
**我想要** 直接跳轉到指定頁碼  
**以便** 快速找到想看的文章頁面

**驗收條件**：
- [x] 提供「到第 X 頁」輸入框
- [x] 提供「送出」按鈕
- [x] 輸入頁碼需在有效範圍內（1 到總頁數），無法輸入無效頁碼
- [x] 跳轉後正確顯示對應頁面的文章
- [x] 當前頁碼高亮更新
- [x] 改變每頁顯示數量時，自動重置「到第 X 頁」輸入框的值

**優先級**：P1  
**相關功能**：跳轉功能（2.1.3.4）

---

## 響應式設計需求

### US-008：手機版瀏覽體驗
**作為** 使用手機的求職者  
**我想要** 在手機上舒適地瀏覽文章列表  
**以便** 隨時隨地了解公司文化

**驗收條件**：
- [ ] 文章列表在手機上垂直排列
- [ ] 文字大小適中，易於閱讀
- [ ] 按鈕大小適合觸控操作
- [ ] 分頁控制在小螢幕上正常顯示
- [ ] 無水平滾動條

**優先級**：P0（響應式需求）

---

### US-009：平板版瀏覽體驗
**作為** 使用平板的訪客  
**我想要** 在平板上獲得良好的瀏覽體驗  
**以便** 充分利用螢幕空間

**驗收條件**：
- [ ] 佈局在平板尺寸（768px - 1024px）上正常顯示
- [ ] 文字與間距適中
- [ ] 觸控操作流暢

**優先級**：P0（響應式需求）
