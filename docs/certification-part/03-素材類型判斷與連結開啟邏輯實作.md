# 03 素材類型判斷與連結開啟邏輯實作
# 03 Media Type Detection and Link Opening Logic

## 使用者故事 User Story

**作為 As** 網站訪客  
**我想要 I want** 依照素材類型有正確的點擊行為（圖片僅輪播、PDF/影片另開新分頁）  
**以便 So that** 不同內容都能以最適方式閱讀或觀看，且維持瀏覽流程不中斷。

## 輸入格式 Input

- 從 `CertificateMediaItem` 取得的：
  - `sourceUrl`
  - （可能）檔案副檔名或 MIME type
  - 影片連結格式（例如 YouTube、Vimeo 或檔案 URL）
- 使用者行為事件：
  - 點擊當前主素材卡片。

## 輸出格式 Output

- 一組純邏輯函式與行為控制：
  - `detectMediaType(sourceUrl: string): CertificateMediaType`
  - `getClickBehavior(item: CertificateMediaItem): LinkAction`
    - `LinkAction` 包含：
      - `type: 'none' | 'openInNewTab'`
      - `href?: string`
      - `target?: '_blank'`
      - `rel?: string`（例如 `noopener noreferrer`）
- 可整合至上層容器或 UI 元件，用來決定是否在新分頁開啟連結。

## 任務內容 Task Description

- 根據 PRD 規則實作素材類型判斷：
  - 內容來源為圖片檔案 → `image`
  - 內容來源為 PDF 檔案 → `pdf`
  - 內容來源為影片連結 → `video`
  - 其他類型 → 排除 / 不在本次範圍。
- 實作點擊主素材時的決策邏輯（統一不用彈窗、改採另開連結）：
  - `image`：
    - 回傳 `type: 'none'`，不做任何開啟行為，由輪播本身切換瀏覽。
  - `pdf`：
    - 回傳 `type: 'openInNewTab'`，附帶 `href = sourceUrl`、`target = '_blank'`、`rel = 'noopener noreferrer'`。
  - `video`：
    - 回傳 `type: 'openInNewTab'`，附帶 `href = sourceUrl`、`target = '_blank'`、`rel = 'noopener noreferrer'`。
- 提供簡單的 helper，讓 UI 元件可以很容易包成 `<a>` 或在 click handler 中 `window.open(...)`。

## 驗收條件 Acceptance Criteria

- [x] `detectMediaType` 能依據 `sourceUrl` 正確區分圖片、PDF 與影片連結。
- [x] 非圖片 / PDF / 影片來源不會被納入顯示或互動流程。
- [x] 對於圖片素材，點擊不會觸發任何新分頁或彈窗行為，只透過輪播瀏覽。
- [x] 對於 PDF 素材，點擊會以 `target="_blank"` 另開新分頁顯示 PDF。
- [x] 對於影片素材，點擊會以 `target="_blank"` 另開新分頁播放影片內容。
- [x] 提供的 `LinkAction` 結構足以被 React 元件直接用於 `<a>` tag 或 click handler。
- [x] 可透過簡單單元測試驗證三種素材類型的邏輯分支與 target 設定。
