# 04 前後台串接與專業證照區塊整合
# 04 Backend Integration and Certificate Section Composition

## 使用者故事 User Story

**作為 As** 網站訪客  
**我想要 I want** 在頁面上看到完整可操作的專業證照區塊  
**以便 So that** 能順暢瀏覽所有證照圖片，並在需要時以新分頁查看 PDF 或影片內容。

### 實作情境 Implementation Context

- **整合位置 Placement**：專業證照區塊（`CertificateSection`）整合於 `src/app/about` 頁面最底下，作為最後一個主要內容區塊（位於 `RelatedIndustriesSection` 之後）。
- **資料來源 Data Source**：透過 GraphQL 串接 CMS 資料，相關實作位於 `src/lib/graphql.ts`（約 L308–L393）：
  - `CertificationGqlItem`、`CertificationsResponse`：GraphQL 回應型別
  - `GET_CERTIFICATIONS`：查詢語句
  - `transformCertificationsToCertificateMediaItems`：將 GraphQL 結果轉成 `CertificateMediaItem[]`
  - `fetchCertificateMediaItems()`：對外使用的資料取得函式
- **現階段策略 Current Phase**：先以 mock data 先行開發與驗證 UI，待 CMS 端準備完成後再切換至 `csrClient.query` 實際查詢。

## 輸入格式 Input

- Strapi CMS API endpoint（URL + 查詢參數）。
- 前端頁面容器（Next.js page / section）。
- 前面任務完成後提供的：
  - 型別與轉換函式（`CertificateMediaItem`、`convertCertificateMedia`）。
  - 輪播 UI 元件（`CertificateCarousel`）。
  - 點擊行為與連結開啟邏輯（`detectMediaType`、`getClickBehavior`）。

## 輸出格式 Output

- 一個整合後的「專業證照」區塊容器元件（例如 `CertificateSection`）：
  - 從後端抓取資料 → normalize。
  - 傳入 `CertificateCarousel` 做顯示。
  - 將主素材卡片包成可點擊的 `<a>`（或等效行為），對 PDF/影片採用 `target="_blank"`。
  - 處理 Loading / Error 狀態（依專案慣例）。

## 任務內容 Task Description

- 實作資料取得層：
  - 呼叫 Strapi 專用 API，取得專業證照素材列表。
  - 使用 `convertCertificateMedia` 轉換成 `CertificateMediaItem[]`。
- 實作主容器元件：
  - 管理 `activeIndex` 狀態。
  - 將資料與事件綁定到 `CertificateCarousel`。
  - 對當前主素材套用 `getClickBehavior`：
    - 若回傳 `type: 'none'`：不新增連結，只提供輪播瀏覽。
    - 若回傳 `type: 'openInNewTab'`：將主素材外層包成 `<a href=... target="_blank" rel="noopener noreferrer">` 或在 click handler 中 `window.open(...)`。
- 與頁面其他區塊整合（標題、段落、整體 section 間距）。

## 驗收條件 Acceptance Criteria

- [ ] 從後端取得的資料能正確顯示於輪播區塊。
- [ ] 每一筆素材皆包含預覽圖、內容來源與素材名稱三項必填資料，缺一則忽略或防禦處理。
- [ ] 前台輪播一律以預覽圖作為入口顯示，不因素材類型不同而改變卡片顯示方式。
- [ ] 使用者可透過滑動或箭頭按鈕切換前一筆 / 下一筆素材。
- [ ] 輪播進度指示器可正確對應目前素材位置。
- [ ] 圖片素材瀏覽流程：
  - 僅使用輪播切換，不會開啟新分頁或彈出視窗。
- [ ] PDF 素材點擊後：
  - 會以 `target="_blank"` 另開新分頁顯示 PDF 內容。
- [ ] 影片素材點擊後：
  - 會以 `target="_blank"` 另開新分頁播放影片內容。
- [ ] 不再出現任何 PDF / 影片彈出視窗的實作或依賴。
