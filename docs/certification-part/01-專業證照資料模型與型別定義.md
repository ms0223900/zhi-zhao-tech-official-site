# 01 專業證照資料模型與型別定義
# 01 Certificate Media Data Model and Types

## 使用者故事 User Story

**作為 As** 前端開發者  
**我想要 I want** 有明確的「專業證照素材」型別定義與資料模型  
**以便 So that** 能安全地從 CMS 串接並在前端統一處理顯示與互動邏輯。

## 輸入格式 Input

- 來源：Strapi CMS API 回傳的 JSON（含專業證照素材列表）。
- 每筆素材至少包含：
  - `previewImage`：預覽圖媒體物件或 URL。
  - `source`：內容來源（圖片檔 / PDF 檔 / 影片連結）。
  - `name`：素材名稱字串。
- 可能有多餘欄位（非本次規格範圍）。

## 輸出格式 Output

- 在前端程式中定義完成的 TypeScript 型別：
  - `CertificateMediaType`：`'image' | 'pdf' | 'video'`
  - `CertificateMediaItem`：包含
    - `id`
    - `previewImageUrl`
    - `sourceUrl`
    - `name`
    - `mediaType: CertificateMediaType`
- 專用的 converter function：
  - `convertCertificateMedia(raw: unknown): CertificateMediaItem[]`

## 任務內容 Task Description

- 設計並撰寫 TypeScript 型別與介面，用於描述單筆「專業證照素材」資料。
- 根據 PRD 的「預覽圖 / 內容來源 / 素材名稱」三個必填欄位定義必要欄位。
- 定義 `CertificateMediaType` 常數或型別，以利後續依素材類型套用不同互動邏輯。
- 規劃並撰寫一個 converter function，將 CMS 回傳的 raw JSON 轉換為 `CertificateMediaItem[]`。

## 驗收條件 Acceptance Criteria

- [x] TypeScript 型別在專案內可被匯入並通過編譯。
- [x] 型別中明確區分 `previewImageUrl`、`sourceUrl`、`name`、`mediaType`。
- [x] `CertificateMediaType` 僅允許 `'image' | 'pdf' | 'video'` 三種值。
- [x] `convertCertificateMedia` 對不合法輸入有基本防禦（例如忽略不完整資料或回傳空陣列）。
- [x] 所有使用該型別的檔案透過 TypeScript 檢查無錯誤。

## 實作說明 Implementation Notes

- **程式碼位置**：`types/certificate-media/`（型別、Entity、轉換邏輯模組化）
- **匯入範例**：
  ```typescript
  import {
    CertificateMediaType,
    CertificateMediaItem,
    CertificateMediaEntity,
    CERTIFICATE_MEDIA_TYPES,
    convertCertificateMedia,
    detectMediaType,
  } from "@/types/certificate-media";
  ```
- **OOP Entity**：`CertificateMediaEntity` 封裝領域邏輯，含 `fromRaw()` 工廠、`isImage`/`isPdf`/`isVideo` getter，及 `toPlainObject()` 供既有 API 相容。
- **Strapi 結構支援**：`convertCertificateMedia` 接受陣列、`{ data: [...] }` 或 `{ attributes: { items: [...] } }` 格式。
- **防禦邏輯**：缺 previewImage、source、name 或無法判斷 mediaType 的項目會被略過。
- **detectMediaType**：依 `sourceUrl` 副檔名與網域（YouTube、Vimeo）判斷類型，供任務 03 的 `getClickBehavior` 使用。
